升级到Xcode 7和IOS9带来的一些问题

1.无法访问http协议的地址

  • 错误信息：

The resource could not be loaded because the App Transport Security
policy requires the use of a secure connection.

  • 解决办法：

iOS9让所有的HTTP默认使用了HTTPS，原来的HTTP协议传输都改成TLS1.2协议进行传输。直接造成的情况就是App发请求的时候弹出网络无法连接。解决办法就是在项目的info.plist
文件里加上如下节点： NSAppTransportSecurity - NSAllowsArbitraryLoads
这个子节点的意思是：是否允许任性的加载？！
设为YES的话就将禁用了AppTransportSecurity转而使用用户自定义的设置，这个问题就解决了。


2.运行时bitcode报错

  • 错误信息：

XXXX’ does not contain bitcode. You must rebuild it with bitcode
enabled (Xcode setting ENABLE_BITCODE), obtain an updated library from
the vendor, or disable bitcode for this target. for architecture arm64

  • 解决办法：

禁用bitcode，选为NO.(iOS中bitcode是默认YES，watchOS中bitcodes是不让改的必须YES。

问题的原因：bitcode的理解应该是把程序编译成的一种过渡代码，然后苹果再把这个过渡代码编译成可执行的程序。bitcode也允许苹果在后期重新优化我们程序的二进制文件，有类似于App瘦身的思想。某些第三方库还不支持bitcode。要不然是等待库的开发者升级了此项功能我们更新库，要不就是把这个bitcode禁用。

Github默认域名绑定自己的域名

  • 在项目根目录下建立一文件，名为CNAME，里面写上域名，如app.com 或者在命令行里执行：

$ echo ‘app.com’ >> source/CNAME

  • 去自己的域名运营商（我是name.com）处修改:

CNAME: www.app.com => wjie2014.github.com

  • 注意：文中app请替换为自己的域名,wjie2014替换为自己的github名。
2015-11-12

在hexo的博客系统中写一篇文章并发布到github

（一）下面是写文章的一种方式

1.进去到项目的source/_posts文件夹，创建文件如：我的一篇文章.md。

cd source/_posts

touch 我的一篇文章.md

2.hexo是支持markdown，使用在线markdown编辑器写好文章，全选文章copy，打开刚刚创建的文件，粘贴，保存。在浏览器上即可访问到刚刚添加的文章了。

附：在线markdown地址：https://www.zybuluo.com/mdeditor#211364

（二）发布hexo文章到github

1.创建github仓库

  • 名字必须是: your_github_name.github.io
  • 例如我的github: wjie2014.github.io
    wjie2014是我的github账户的名称，必须注意这个，不要写错，否则是看不到发布的博客的。

2.修改_config.yml

  • 编辑全局配置文件 _config.yml 中的 deploy 部分

deploy:

type: git

repo: git@github.com:wjie2014/wjie2014.github.io.git

branch: master

3.部署

hexo clean//清理数据缓存，建议每次都清楚

hexo d //部署

  • 部署成功的信息

[info] Deploy done: github

  • 部署失败的信息

ERROR Deployer not found: git

  • 解决办法：

控制台依次运行命令

npm install hexo-deployer-git –save

hexo g

hexo d

4.在浏览器输入http://wjie2014.github.io/就能看到网页了。
